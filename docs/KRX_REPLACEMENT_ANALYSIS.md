- ✅ 종목명 (ISU_NM)

- ✅ 등락률 (FLUC_RT)
- ❌ **기술적 지표 없음** (RSI, MACD, 이동평균 등)

## 🚨 KRX API 한계점

### 1. 종목 수 차이
```
현재: 3,560개 (KOSPI + KOSDAQ)
KRX:   962개 (KOSPI만)
차이: 2,598개 (73% 감소)
```

**원인 추정**:
- KRX는 유가증권(보통주)만 제공
- 현재 시스템은 우선주, ETF, ETN, 스팩 등 포함
- KOSDAQ은 별도 API 권한 필요

### 2. 기술적 지표 부재
KRX API는 원시 가격 데이터만 제공:
- ❌ RSI (상대강도지수)
- ❌ MACD (이동평균수렴확산)
- ❌ 이동평균선 (MA5, MA20, MA60)
- ❌ 볼린저밴드
- ❌ ATR (평균진폭)

**→ 별도 계산 필요** (pandas-ta, talib 등)

### 3. API 호출 한도
- 일일 10,000회 제한
- 3,500종목 × 1일 = 3,500회 (종목당 1회 호출 시)
- **최대 2.8일치** 역추적 가능

## 💡 결론 및 제안

### KRX API 단독 사용: ❌ 비추천
이유:
1. 종목 수가 현재의 27% 수준 (962/3,560)
2. KOSDAQ 데이터 없음 (별도 신청 필요)
3. 기술적 지표 모두 재계산 필요
4. 과거 데이터 역추적 불가능 (일일 10,000회 한도)

### 하이브리드 방식: ✅ 추천
```
┌─────────────────────────────────────────────────────┐
│              하이브리드 데이터 구조                 │
├─────────────────────────────────────────────────────┤
│  KRX API                    FDR (기존)            │
│  ─────────                  ───────────           │
│  • 당일/전일 데이터         • 과거 데이터         │
│  • 지수 데이터              • 기술적 지표         │
│  • 실시간성 보장            • 전 종목 커버리지    │
│  • 빠른 속도                • KOSDAQ 포함         │
└─────────────────────────────────────────────────────┘
```

### 구체적 적용 방안

#### 단기 (1-2주)
1. **KRX API를 "당일 데이터 보정"용으로 사용**

   - 4/6, 4/7 데이터 누락분 KRX로 보충

2. **지수 데이터는 KRX로 전환**
   - KOSPI/KOSDAQ 지수: KRX API 사용 (이미 승인됨)
   - 더 빠르고 안정적

#### 중기 (1-2개월)
1. **KOSDAQ API 권한 추가 신청**
   - 코스닥 일별매매정보 신청
   - KOSDAQ 2,500+ 종목 커버리지 확보

2. **기술적 지표 계산 모듈 개발**
   - KRX 원시 데이터 → RSI, MACD, MA 계산
   - `v2/core/indicators.py` 확장

#### 장기 (3개월+)
1. **실시간 데이터 연동 검토**
   - KRX 실시간 시세 API (WebSocket)
   - WebSocket + KRX API 조합

2. **데이터 아키텍처 개편**
   - Raw Data (KRX) → Indicator Engine → Scanner
   - DB 스키마: 원시 데이터 + 계산 지표 분리 저장

## 🎯 즉시 적용 가능한 개선안

### 개선안 1: 당일 데이터 보정 스크립트
```python
from v2.core.krx_datasource import KRXDataSource
from v2.core.data_manager import DataManager

# KRX로 누락된 데이터 보충
source = KRXDataSource()
df = source.fetch_daily_prices('20250407')  # KOSPI 962개
dm = DataManager()
dm.upsert_prices(df)  # 기존 DB에 병합
```

### 개선안 2: 이중 소스 패턴
```python
def get_price_data(code, date):
    # 1. 기존 DB 확인
    price = db.get_price(code, date)
    if price:
        return price
    
    # 2. KRX API 폴백
    price = krx_api.get_stock_detail(code, date)
    if price:
        db.save_price(price)
        return price
    
    # 3. FinanceDataReader 폴백
    return fdr.get_price(code, date)
```

## 결론

**KRX API 단독으로는 현재 시스템을 대체하기 어려움.**

그러나:
- ✅ **지수 데이터**는 KRX로 대체 권장 (빠르고 안정적)
- ✅ **당일 데이터 보정**은 KRX로 활용 권장 (실패 시 폴백)
- ✅ **KOSDAQ 권한 추가** 시 활용도 크게 향상
- ⚠️ **과거 데이터 + 기술적 지표**는 기존 FDR 유지 필요
