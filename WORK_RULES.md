# WORK_RULES.md - 작업 원칙

## 🚫 절대 수정하지 않는 항목

### 1. 백테스트 기반 기준
- **70점 이상**: 승률 80% (적극 매수)
- **60-69점**: 승률 40% (참고용)
- **60점 미만**: 제외

> ⚠️ **중요**: 사용자가 명시적으로 지시하지 않는 한 절대 수정 금지

### 2. 보유 기간
- **3일 이상 보유**: 승률 64.3%
- **1일 스윙**: 승률 45.2% (비권장)

### 3. 리스크 관리
- **개별 종목**: 최대 3% 포지션
- **손절가**: -3%
- **익절가**: +5%

- 알림 설정

## 🔄 리포트 생성 및 배포 프로세스

### 필수 절차 (모든 리포트 대상)

```
┌─────────────────────────────────────────────────────────────┐
│  1. 생성  →  2. 검토  →  3. 개선  →  4. 재검토  →  5. 배포  │
└─────────────────────────────────────────────────────────────┘
```

#### 1️⃣ 리포트 생성
- 스캐너 실행 및 결과 수집
- HTML/JSON 리포트 파일 생성
- 로컬 저장 (`docs/` 디렉토리)

#### 2️⃣ 품질 검토 (필수)
**데이터 정확성**
- [ ] 종목명 인코딩 확인 (한글 깨짐 여부)
- [ ] 종목 코드 매칭 확인
- [ ] 가격 데이터 최신성 확인
- [ ] 점수 계산 로직 검증

**리포트 형식**
- [ ] HTML 렌더링 정상 확인
- [ ] 모바일/데스크톱 반응형 확인
- [ ] 테이블/차트 정렬 확인

**논리적 일관성**
- [ ] 등급 분류 기준 준수 (70+/60-69/60-)
- [ ] 승률 표기 정확성
- [ ] 매매 기준 표기 정확성

#### 3️⃣ 문제 발견 시 개선
- 인코딩 문제 → `utf-8` 명시적 설정
- 데이터 오류 → 재스캔 또는 보정
- UI 문제 → CSS/HTML 수정

#### 4️⃣ 재검토
- 개선사항 반영 후 재확인
- 최종 품질 체크리스트 통과

#### 5️⃣ GitHub 배포
```bash
# 1. docs/ 디렉토리 파일 확인
ls -la docs/

# 2. 루트에 복사 (GitHub Pages용)
cp docs/REPORT_NAME.html .

# 3. Git 작업
git add -A
git commit -m "type: Description

- Change 1
- Change 2"
git push origin main

# 4. 배포 확인
# https://lubanana.github.io/kstock-dashboard/REPORT_NAME.html
```

### ⚠️ 배포 금지 사항
- [ ] 종목명 깨짐 현상 발견 시
- [ ] 점수 계산 오류 발견 시
- [ ] 데이터 누락 발견 시
- [ ] HTML 깨짐 발견 시

> **핵심 원칙**: 완벽하지 않으면 배포하지 않는다.
