## 1️⃣ Pivot Point Strategy (피벗 포인트 돌파 전략)

### 현재 상태
- **진입:** 20일/60일 신고가 돌파 + 거래량 150% 이상
- **피라미딩:** 30% → 30% → 40% (5% 상승마다 추가 진입)
- **손절:** 평균단가 -10%
- **익절:** 최고점 대비 -15% (트레일링 스탑)

### 🔴 개선 필요 사항

#### 1.1 피라미딩 타이밍 개선
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 5% 상승마다 추가 진입 | 상승 초기에 포지션 과다 집중 | ATR(평균진폭) 기반 진입 |
| 고정 5% | 변동성 무시 | 변동성이 클 때는 8%, 작을 때는 3% |

```python
# 개선안: ATR 기반 피라미딩
atr = df['high'].rolling(14).max() - df['low'].rolling(14).min()
pyramid_step = atr / current_price * 100  # ATR %로 계산
```

#### 1.2 손절 라인 동적 조정
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 고정 -10% | 변동성 큰 종목은 너무 빡빡, 작은 종목은 너무 느슨 | ATR 기반 손절 (-2ATR) |

```python
# 개선안: ATR 기반 손절
stop_loss_price = entry_price - (2 * atr)
# 예: ATR 500원 → 손절가 -1,000원 (-5% @ 20,000원)
# 예: ATR 2,000원 → 손절가 -4,000원 (-5% @ 80,000원)
```

#### 1.3 거래량 조건 강화
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 거래량 150% 이상 | 평소 거래량이 적은 종목도 진입 | 최소 거래대금 조건 추가 |

```python
# 개선안: 거래대금 + 거래량 동시 조건
if volume_ratio > 1.5 and amount > 10_000_000_000:  # 100억 이상
    signal = True
```

#### 1.4 RSI 필터 추가
- **문제:** 과매수 구간에서 진입 시 손실 확률 증가
- **개선안:** RSI > 70이면 진입 제한 (과매수 회피)

```python
if rsi > 70:
    return False  # 과매수 구간 진입 금지
```

## 2️⃣ Buy Strength Score (매수 강도 점수)

### 현재 상태
- **추세 (40점):** 정배열 + 눌림목
- **모멘텀 (30점):** RSI 50~70 + MACD Histogram 증가
- **거래량 (30점):** 20일 평균 대비 거래량 비율
- **하드 필터:** 거래대금 10억+, 주가 1,000원+

### 🔴 개선 필요 사항

#### 2.1 RSI 범위 조정
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 50~70 | 70 근처는 이미 과매수 | 45~65로 하향 조정 |
| 고정 범위 | 강세장/약세장 무시 | 시장 상황별 동적 범위 |

```python
# 개선안: 시장 상황별 RSI 조정
if market_trend == 'bull':
    rsi_range = (50, 75)  # 강세장은 높은 RSI 허용
else:
    rsi_range = (40, 60)  # 약세장은 낮은 RSI 선호
```

#### 2.2 거래량 점수 세분화
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 200%=30점, 150%=20점, 100%=10점 | 300% 이상은 과열 신호 | 상한선 추가, 과열 패널티 |

```python
# 개선안: 과거 대비 거래량도 고려
if volume_ratio > 300 and price_change > 10:
    volume_score = 20  # 과열 시 점수 감점
else:
    volume_score = min(30, volume_ratio / 200 * 30)
```

#### 2.3 눌림목 정의 개선
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 20일선 대비 0~3% | 너무 좁은 범위 | 20일선 대비 -2%~+5%로 확대 |
| 단순 % 비교 | 볼린저 밴드 활용 안 함 | 볼린저 하단 근접 시 가산점 |

```python
# 개선안: 볼린저 밴드 활용
bb_lower = sma20 - (2 * std20)
if current_price < bb_lower * 1.02:  # 하단 +2% 이내
    trend_score += 10  # 추가 점수
```

#### 2.4 섹터/테마 가중치
- **문제:** 모든 종목 동일 기준 적용
- **개선안:** 유망 섹터(반도체, 바이오)에 가중치 부여

```python
sector_bonus = {
```

## 3️⃣ SEPA + VCP Strategy (Mark Minervini)

### 현재 상태
- **Trend Template:** 8가지 조건 (MA150/200, 52주 고저, RS 상위 25%)
- **VCP Pattern:** 2~4회 수축, 깊이/기간 점진적 감소
- **Volume:** Dry-up (60% 이하) 후 Surge (150% 이상)

### 🔴 개선 필요 사항

#### 3.1 VCP 수축 횟수 유연화
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 2~4회 고정 | 5회 이상 수축도 우량 패턴 | 2~6회로 확대 |
| 모든 수축 동일 가중 | 마지막 수축이 가장 중요 | 마지막 수축에 가중치 부여 |

```python
# 개선안: 가중치 부여
if contraction_count >= 2:
    # 마지막 수축이 가장 중요
    last_contraction_weight = 0.5
    others_weight = 0.5 / (contraction_count - 1)
```

#### 3.2 RS (Relative Strength) 계산 개선
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 단순 상위 25% | 코스피/코스닥 구분 없음 | 각 시장별 상위 25% |
| 현재 시점만 비교 | 추세 고려 안 함 | RS 상승 추세 추가 조건 |

```python
# 개선안: RS 추세 확인
rs_current = price / market_index
rs_20days_ago = price_20d / market_index_20d
if rs_current > rs_20days_ago * 1.05:  # RS 상승 추세
    rs_score += 10
```

#### 3.3 피벗 돌파 확인 시간 추가
| 현재 | 문제점 | 개선안 |
|------|--------|--------|
| 당일 돌파만 확인 | False breakout 다수 | 3일 내 확인 조건 |
| 종가만 확인 | 장중 돌파 후 하락 | 종가 기준 확인 |

```python
# 개선안: 3일 confirmation
breakout_count = 0
for i in range(3):
    if df.iloc[-(i+1)]['close'] > pivot_line:
        breakout_count += 1
if breakout_count >= 2:  # 3일 중 2일 이상 돌파
    signal = True
```

#### 3.4 섹터 RS 추가
- **문제:** 개별 종목 RS만 고려
- **개선안:** 소속 섹터의 RS도 함께 확인

```python
# 섹터 RS가 상위 50% + 개별 RS 상위 25%
if sector_rs_rank < 0.5 and stock_rs_rank < 0.25:
    signal_strength += 1
```

## 5️⃣ 공통 개선 사항

### 5.1 포지션 사이징 개선
| 현재 | 개선안 |
|------|--------|
| 고정 비율 | 켈리 공식 기반 동적 사이징 |
| 모든 종목 동일 | 전략 개수별 차등 (1개=5%, 2개=10%, 3개=15%) |

```python
# 켈리 공식: f = (p*b - q) / b
# p: 승률, q: 패률, b: 평균 수익/평균 손실
win_rate = 0.387  # 승률
avg_win = 0.158   # 평균 수익
avg_loss = 0.0636 # 평균 손실
b = avg_win / avg_loss  # 2.48
q = 1 - win_rate  # 0.613

kelly = (win_rate * b - q) / b  # 약 0.15 (15%)
position_size = min(kelly, 0.15)  # 최대 15% 제한
```
